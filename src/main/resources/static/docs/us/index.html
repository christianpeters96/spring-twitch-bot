    Introduction

    Welcome to the SharpaBot documentation! This documentation explains how to use the Twitch bot, how custom commands work (what variables and methods are available, and usage examples), and explains the individual endpoints of our API.

    The dark area on the right side shows code examples for the bot commands, as well as examples and return values for the API request.

    This documentation was created using Slate.

    Using the website

    Login

    For authentication, OAuth 2.0 is used. To log in, just press "Connect to Twitch". You will be forwarded to an external website (id.twitch.tv). If you are already logged in to Twitch, you only need to authorize the service for SharpaBot. If you are not yet logged in, you have to log in with your Twitch data.

    web interface

    As soon as you have logged in, you should land on the dashboard. On the left side you see the navigation, with which you can switch to the different subpages (e.g. commands or security).

    At the top of the bar you can set the language and see the current status of your twitch bot (Connected, Not Connected). With a click on "Connect" you can start your bot - provided that the authentication token of your bot is already set. If the bot has already been started, you can shut down the bot by clicking on "Disconnect".

    Further left you see the bell. It shows notifications when you click on it. The red number next to the bell indicates whether there are any notifications and if so, how many (if no number is visible, there are none).

    Page: Authentication

    This page is used to set the authentication token for the bot. You can get the token here. This token must be set, otherwise the twitch bot cannot connect.

    For some functions additional services must be linked. For example, because the song requests run via Spotify, a Spotify link is needed. Just click on "Link Spotify".

    You can get to this page by clicking on your avatar (picture above right) and clicking on "Authentication"

    Page: Dashboard

    On the dashboard you can view stream data of the last 7 days. This includes the number of sent messages, sent commands, deleted rewrites and distributed timeouts. Next to it a line chart can be seen, which visualizes the values of the individual days. Below the blue and red text there are two grey numbers. The first number shows the difference to the last stream and the second number (in brackets) shows the percentage increase (or decrease).

    Below this are 2 blue boxes showing the Top Chatter (the user who was most active in the chat) and the Top Command (most used command) of the last 7 days.

    Page: Commands

    On the page "Commands" you can see a table which contains all ready created commands. By default the bot has no commands, so you have to create them first. To do this, simply press "Create".

    If you press "Create" you will get to a new page where you can set the parameters of the command. You only need 3 fields -> name, command and the action (well, there can be several fields).

    Name (Information)

    The name of the command. This is a meta information and is not used in the stream. The name is displayed on this web page in the command table to identify the command. At some point there will be a table for all users by URL (as known from Nightbot, Moobot or other bots; then the name will not be as irrelevant.

    Description (Information)

    The same applies to the description as to the name, but the description is not displayed on the command management table. Later in the public table the description will be shown.

    command (chat usage)

    The command specifies what string the user must write in the streamchat to trigger the command. (for example "!age" or "!elo")

    Alias (chat usage)

    If "Add Alias" is pressed, any number of alternative commands can be specified. (for example "!songrequest" (command) and "!sr" (alias))

    Global (delay)

    The global delay is used to prevent commands from being spammed. It is specified in seconds. If the delay is set to 30, the command cannot be reused until 30 seconds after it was last used.

    Per user (delay)

    For the "Per User" delay, the same applies to the global delay, except that the global delay applies only to the user who used the command. This prevents the same user from using the command more than once and other users from not getting the chance to do so (e.g. for song requests you can set a global delay of 10 seconds and a "Per User" delay of 30 seconds, so that the same user does not always add his songs).

    activity points (costs)

    This specifies whether the command costs activity points. The user's activity points can only be viewed if a command is created that outputs them.

    Create a new command with the action "Chat input" and enter in the message for example "You have ${points} Activity points")

    Actions

    Here you can choose templates or give direct commands to the bot (template "Raw"). Some templates have predefined default values (like "Song Request" or "Current Song"), so they can be created quickly (because there are no standard commands)

    Rights

    Here you can define who is allowed to use the command. You can select a role, or give the right to selected users. As a broadcaster you can always use the command. If you want the command to be available for everyone, leave the table empty.

    Page: Security -> Spam protection

    On this page you can set your spam protection. By default all types of spam protection are disabled. With the "Activate" button these can be activated. But don't forget to save after activation (by clicking the "Save" button).

    Blacklist

    The blacklist filter is a simple word or string filter. You can enter words or sentences, which you don't want to see in your stream. If the string is found in a message, the user who created it gets a timeout.

    Caps-Lock

    The "Caps Lock" filter is a filter, which prevents the excessive use of caps lock messages (ALL IN GROẞBUCHSTABEN). You can set a limit (8 by default). If the limit is exceeded within 5 minutes, the user gets a timeout.

    Emotes

    The "Emote"-filter works exactly the same way as the "Caps-Lock"-filter, only here messages are taken, which contain 4 or more emotes.

    Links

    Simple thing - If someone posts a link, there is a timeout (whitelist function is in planning, for clip whitelist etc.)

    Special characters

    The special character filter works almost like the "Caps-Lock" filter, but the rules are a bit stricter. If the message consists of 25% of special characters, the message is considered potential spam. If the message consists of 50% special characters, it is even counted twice.

    Links

    We will try to integrate as many services as possible in this project. Currently Spotify and Discord are available. By clicking on your avatar (picture above right) and "Authentication" you will get to the page to create these links.

    Spotify

    Link Spotify to enable the following features:

    • Song Request (You can request a song)
    • Song request (which song is currently playing)
    • (Planned) Song Skip

    Discord

    Link Discord to enable the following features:

    • Discord-Webhook (notification when you go live)

    Commands: Functions

    To give the broadcaster as much freedom as possible when creating commands, there are the possibilities to tell the bot directly what to do when a command is entered (raw input). But to make the creation not too complicated at the same time, standard templates were built in, where you only have to fill in the necessary fields. However, the raw functions are described in more detail here.

    There is also a number field above, where you can specify how many parameters your created command has. How to use the parameters is explained in User defined arguments.

    Function: 'set'

    If you want to change the message when a song request happens, but Spotify is not running (or paused), you only have to change the variable 'potify.paused'.

    set spotify.paused Sorry, nothing is currently running here.
# Parameter 1: "spotify.paused"
# Parameter 2: "Sorry, nothing is currently running here."

    If you now use the Song Request method, the newly defined 'pause message' is used instead of the default message.

    The function 'set' is relatively simple. It expects 2 parameters.

    The first parameter is the unique key or the name of the variable. This is needed to be able to use the variable later or to allow the system to recognize and assign the variable (which is more likely).

    The second parameter is the value or content of the variable.

    Both the key and the value can consist of any characters. While the key may not contain spaces, the value may contain any number of spaces. Variables are mostly optional, because there are default values for most cases (current ones even in all cases).

    Function: 'say'

    Example: The Bot Greets

    say Hello HeyGuys

    This is probably the easiest function of all. It needs 1 parameter.

    The parameter is a string of any length and all characters are allowed (as long as they work in twitch chat).

    You can also use variables.

    Function: 'spotify.request'

    Add Sandstorm from Darude and set all (optional) variables.

    set spotify.success Next you can enjoy '${song.name}' from ${song.artist}.
set spotify.error Not wanting to right now. Try again later.
set spotify.paused Currently nothing is playing. Try again later.
spotify.request Darude Sandstorm
say ${response}

    If all goes well, the bot should now answer: 'Next you can enjoy 'Sandstorm' by Darude.`

    It's possible that songs have more than one artist, in this case they are automatically separated (with comma + 'and').

    Example: 'Next you can enjoy 'Forever' by Justin Bieber, Post Malone and Clever.`

    The function searches for a song (via Spotify) and adds the found song to the queue. This only works if Spotify is running and not paused. The command needs 1 parameter and uses 3 optional variables.

    The parameter is the search string. The variables are messages which are output in different cases.

    • Message if everything worked. (spotify.success)
    • Message when something unexpected went wrong (server side problems or similar) (spotify.error)
    • Message when Spotify is not running or paused. (spotify.paused)

    After the function spotify.request is executed, the bot sets a variable response in which it enters the message (one of the 3 cases mentioned above).

    The function also creates the variables song.name (song title) and song.artist (artist (can be several)).

    variables are used by setting a dollar sign and entering the name of the variable in curly brackets. Example: ${song.name}

    Function: 'spotify.get'

    Get current song information with user defined answers:

    set spotify.success We are listening to '${song.name}' from ${song.artist}
set spotify.paused Nope, is pause.
spotify.get
say ${response}

    The function returns information about the currently played song (via Spotify) and writes the information to a variable. This only works if Spotify is running and not paused. The command needs no parameters and uses 2 optional variables.

    The variables are messages that are output in different cases.

    • Message when everything has worked. (spotify.success)
    • Message when Spotify is not running or paused. (spotify.paused)

    After the function 'spotify.get' has been executed, the bot sets a variable response in which it enters the message (one of the 2 cases mentioned above).

    The function also creates the variables song.name (song title) and song.artist (artist (can be several)).

    Commands: Variables

    To output information which is not static (like the name, the origin or similar) you can use variables. We try to offer as many variables as possible so that you can personalize your commands as much as possible.

    Variables are accessed by using a dollar sign and then entering the name of the variable in curly brackets. Do not use spaces in these brackets.

    Variables can be created using the 'set' function. But there are also other variables. This section explains what they are and how to use them.

    User defined arguments

    Example: Own timeout command (number of arguments: 3)

    say /timeout ${1} ${2} ${3}

    If your command needs parameters, for example the search term in the song request command, then these parameters are written to variables. The name of the variable corresponds to a number from 1 to infinity. The 1 is the first parameter, the 2 is the second and so on.

    Variables created by the user

    Example: Command that prints the age

    set age 23
say I am ${age} years old

    After creating a variable with the 'set' function, you can also access this variable.

    Dynamically created variables

    Example: Song request with standard answers (number of arguments: 1)

    spotify.request ${1}
say ${response}

    Temporary (dynamic) variables are created after certain functions have been executed.

    • spotify.request -> sets the variables response, song.name and song.artist
    • spotify.get -> sets the variables response, song.name and song.artist

    Variable: 'user'

    Example: The bot probably doesn't like the user

    say I find @${user} stupid.
# Output: I find @username stupid.

    The user variable prints the name of the user who executed the command.

    Variable: 'points'

    Example: Outputs the activity points of the user

    say Activity points: ${points}
# Output: Activity points: 37

    The variable points displays the activity points of the user who executed the command.

    By the way: 50 activity points are generated per hour.
